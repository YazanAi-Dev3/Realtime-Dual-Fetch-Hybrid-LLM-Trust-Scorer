# توثيق سيناريوهات محرك التقييم

هذا الملف يوثّق كل السيناريوهات التي يعالجها محرك التقييم في المشروع كما هي مطبقة في `calculate_trust_score` وباقي خط الأنابيب.

---

## 1) كيف تدخل البيانات وتُعالج قبل التقييم

### 1.1 اكتشاف الصفحات
- يتم إنشاء روابط مرشحة جاهزة (سياسات معروفة مثل الخصوصية والاسترجاع).
- يتم استخراج روابط إضافية من الصفحة الرئيسية إذا احتوت كلمات مفتاحية مرتبطة بالسياسات.
- يتم إزالة التكرارات ثم أخذ أول 5 صفحات فقط (MAX_SUBPAGES = 5) حسب الأعلى صلة.

### 1.2 جلب المحتوى (Dual-Fetch)
لكل رابط مرشح:
- جلب أول عبر requests لاستخراج بيانات مخفية داخل script tags.
- جلب ثاني عبر Firecrawl للحصول على Markdown نظيف.
- النص المخصص للـ Regex = Markdown + البيانات المخفية.
- النص المخصص للـ Gemini = Markdown النظيف فقط.
- في حال فشل صفحة معينة: تسجل كخطأ وتكمل بقية الصفحات.

### 1.3 الاستخراج Regex
الحقول المستخرجة:
- phone
- email
- vat_number
- commercial_reg

قواعد إضافية:
- إزالة التكرارات من النتائج.
- البريد يتم توحيده إلى أحرف كبيرة.
- السجل التجاري الذي ينتهي بـ 00000 يتم تجاهله.

### 1.4 الاستخراج Gemini
- يُطلب من Gemini استخراج وجود سياستي الخصوصية والاسترجاع مع ملخص.
- إذا كان النص المجمع قصيرًا جدًا (< 100 حرف): تعاد نتيجة فارغة {}.
- إذا فشل Gemini أو فشل parse JSON: تعاد نتيجة فارغة {}.

## 2) سيناريوهات كل محور تقييم (كل الحالات)

## 2.1 محور الهوية القانونية legal_identity

| الحالة | الشروط | النقاط | ملاحظة |
|---|---|---:|---|
| كاملة | يوجد CR ويوجد VAT | 40 | - |
| جزئية | يوجد واحد فقط من (CR أو VAT) | 25 | تحذير: Partial legal identity |
| غير متوفرة | لا يوجد CR ولا VAT | 0 | - |

## 2.2 محور عمر الدومين domain_longevity

| الحالة | الشرط | النقاط | ملاحظة |
|---|---|---:|---|
| قوي جدًا | العمر >= 3 سنوات | 25 | - |
| جيد | 1 <= العمر < 3 | 15 | - |
| محدود | 0.5 <= العمر < 1 | 10 | - |
| ضعيف/غير معروف | العمر < 0.5 أو فشل WHOIS/التحليل | 0 | قد يظهر تحذير: Domain is very new |

## 2.3 محور قابلية التواصل contactability

| الحالة | الشروط | النقاط | ملاحظة |
|---|---|---:|---|
| ممتاز | يوجد هاتف ويوجد بريد | 15 | - |
| جيد | هاتف فقط | 12 | - |
| محدود | بريد فقط | 5 | - |
| معدوم | لا هاتف ولا بريد | 0 | تحذير: No direct contact methods found |

## 2.4 محور الشفافية transparency

| الحالة | الشروط | النقاط |
|---|---|---:|
| كاملة | privacy_policy.exists = true و refund_policy.exists = true | 20 |
| جزئية 1 | privacy_policy.exists = true و refund_policy.exists = false | 10 |
| جزئية 2 | privacy_policy.exists = false و refund_policy.exists = true | 10 |
| معدومة | الاثنان false أو gemini_data فارغ | 0 |

## 2.5 محور العقوبات penalties

| الحالة | الشرط | العقوبة |
|---|---|---:|
| Ghost Syndrome | domain_age_years < 1 و has_cr = false و has_phone = false | -40 |
| لا عقوبة | غير ذلك | 0 |

ملاحظة دقيقة:
- شرط العقوبة يعتمد على عدم وجود CR والهاتف، ولا يفحص VAT في الشرط.

---

## 3) معادلة النتيجة النهائية

النتيجة الخام:

Score_raw = legal_identity + domain_longevity + contactability + transparency + penalties

النتيجة النهائية بعد التقييد:

total_score = max(0, min(100, Score_raw))

## 4) سيناريوهات اللون/التصنيف النهائي

| المجال | التصنيف | اللون |
|---|---|---|
| 85 إلى 100 | Trusted & Secure | Green |
| 60 إلى 84 | Proceed with Caution | Yellow |
| 0 إلى 59 | High Risk / Suspicious | Red |

---

## 5) كل السيناريوهات المجمعة التي يعتمد عليها التقييم

عدد حالات المحاور قبل العقوبة:
- الهوية القانونية: 3 حالات
- عمر الدومين: 4 حالات
- التواصل: 4 حالات
- الشفافية: 4 حالات

إجمالي التركيبات الأساسية = 3 × 4 × 4 × 4 = 192 تركيبة.

بعد إضافة حالة العقوبة (تُفعل فقط بشرط محدد) تصبح النتيجة:
- نفس 192 تركيبة، وبعضها يطبق عليه -40 عندما يتحقق شرط Ghost Syndrome.

### 5.1 كيف تحسب نتيجة أي تركيبة عمليًا
1. اختر حالة من كل محور في الجداول أعلاه.
2. اجمع نقاطها.
3. افحص شرط العقوبة:
   - إن تحقق: اطرح 40.
   - إن لم يتحقق: لا تغيير.
4. طبّق التقييد بين 0 و100.
5. حدّد اللون من جدول التصنيف.

---

## 6) حالات خاصة مهمة (تشخيصية)

| السيناريو | النتيجة المتوقعة |
|---|---|
| فشل WHOIS بالكامل | domain_longevity = 0 |
| gemini_data فارغ أو فشل JSON | transparency = 0 |
| لا صفحات صالحة للاستخراج | غالبًا regex_data شبه فارغ + gemini_data فارغ |
| متجر جديد (< سنة) بلا CR وبلا هاتف | تفعيل عقوبة Ghost = -40 |
| CR + VAT + عمر >=3 + هاتف + بريد + سياستين | 100/100 |

---

## 7) حالة الدرجة الكاملة 100/100

للوصول إلى 100 يجب تحقق جميع الشروط التالية معًا:
- يوجد CR
- يوجد VAT
- عمر الدومين >= 3 سنوات
- يوجد هاتف
- يوجد بريد
- privacy_policy.exists = true
- refund_policy.exists = true

تفكيك النقاط:
- legal_identity = 40
- domain_longevity = 25
- contactability = 15
- transparency = 20
- penalties = 0

المجموع = 100
